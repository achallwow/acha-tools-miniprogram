# 微信小程序 Vibe Coding 开发日志

## ⚠️ 架构级核心铁律 (ARCHITECTURAL MANDATES)

### 1. 布局坐标锁死 (Layout Freeze) - 最高优先级
- **严禁乱动**: 除非接到明确的“移动/调整间距”指令，否则在实现任何新功能（如拖拽、上传等）时，**绝对禁止**修改已有元素（标题、日期、九宫格容器、标语、品牌签名、按钮）的 CSS 位置属性（top, margin, padding, position, flex 等）。
- **尊重审美资产**: 当前页面的物理坐标是经过精确打磨的“完美状态”，所有功能开发必须在不破坏现有容器流的前提下进行。

### 2. 品牌与视觉规范
- **核心黑 (Apple Black)**: `#1D1D1F`。
- **背景灰 (Apple Gray)**: `#f8f8f8`。
- **系统底色**: 全局 `backgroundColor` 强制锁定为 `#f8f8f8`，消灭转场闪烁。
- **字体**: 标题 `56rpx (weight 800)`, 说明 `24rpx (weight 600)`, 品牌签名 `32rpx (weight 800, letter-spacing 12rpx)`。

### 3. 页面容器规范
- **防遮挡**: 所有页面根节点必须挂载 `.global-page-container` 类，确保避开刘海/灵动岛。
- **全屏锁定**: `.page-container` 必须具备 `min-height: 100vh`，严禁使用超出 `100vw` 的位移黑科技。
- **滚动条消隐**: 废弃位移法，统一使用 `::-webkit-scrollbar { display: none !important; }`。

## 2026-03-04 核心更新记录 (V2.1.0)

### 1. 首页视觉微调
- **焦点推荐**: 高度收敛至 `300rpx`，TOP PICK 标签改为醒目的苹果红（`#FF3B30`）。
- **图标系统**: 实装“安全边界盒”规范，所有图标强制在 `48x48rpx` 盒内通过 4rpx 线条绘制，实现视觉体积绝对统一。
- **工具目录**: 默认显示数量提升至 4 个，统一卡片高度为 `156rpx`。

### 2. 工作日报页面重构
- **画板化设计**: 将标题、日期、标语全部移入白色 `.grid-box` 容器，形成统一的“报表感”。
- **九宫格优化**: 间距从 `8rpx` 精细化收缩至 **`4rpx`**，通过同步增加格子物理尺寸，确保了容器外部坐标的绝对锁定。
- **iOS 级拖拽排序**: 实装基于“坐标映射”的高级流体避让引擎，实现了物理级的滑行碰撞感，且不破坏原始 Flex 布局。
- **全机型自适应**: 底部按钮废弃 `top` 定位，改用 `bottom + env(safe-area-inset-bottom)` 动态锚定，兼容近 5 年所有全面屏/非全面屏手机。
- **视觉稳定性**: 统一了系统底色与页面底色，彻底解决了页面切换时底部灰色横条一闪而过的视觉 Bug。
